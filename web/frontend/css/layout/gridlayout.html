Concepts

  • Grid container
  • Grid items
  • Tracks
    • row tracks: rows
    • columns tracks: columns
  • grid cells: intersection of the tracks (row and columns)
  • grid lines: lines that separate grid cells. It's useful to see them as array indexes, specially when they are used to specify when starts and ends something.

Distribution

  • Start-end
    • We can specify the grid lines where a component starts and ends.
      • Remember: it uses grid lines, not grid cells
        • "grid-column-start: 1 2" //is one cell size, not two
      • We specify both, size and placement.
      • Negative numbers can be used, where -1 is the last grid line.
  • Order
    • By default is 0
    • Elements are added in ascending order.

Size

Cells

  • fixed size: no responsive
  • fixed size + auto
    • auto used in rows will try to expand to 100%.
    • auto used in columns will try to fit its content.
  • fractional: each column/row has a relative size.
  • minmax: define min and max size.

Observations

  • Can be combined
  • We can use value: repeat(N, size) to write the same size N times.

Elements

span: defines how many rows/cells uses an element.

Number of cells vs number of elements

What happens when the number of cells defined by grid layout is different than the number of cells occupied by the elements?

  • Empty cells: unused cells will collapse to a 0 size.
  • Overflow
    • The other elements will use default size values.
    • We can use properties like grid-auto-rows to specify the default values.
