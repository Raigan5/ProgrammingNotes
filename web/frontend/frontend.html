Responsability: simple approach vs components

  • Simple approach: the responsability is separated like in the MVC pattern.
    • Content: HTML
    • View: CSS
    • Behavior: JS
  • Components
    • Each component is responsible of its own content, view and behavior and we can build a web using those components.
    • It's like going from structured programming (simple approach) to OOP.
    • Example: Angular, React.
