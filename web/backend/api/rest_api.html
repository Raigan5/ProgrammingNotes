REST API

General

  • Use HTTP protocol.
  • JSON output.
  • Stateless.
  • Resource-based.

HTTP

Request

Methods

  • GET: request a resource
  • POST: sending a resource
  • PUT: replace a resource
  • PATCH: patch up a resource (replace a part of a resource)
  • DELETE: delete a resource

Format

  • endpoint: baseURL/endpoint
  • query parameters: baseURL/endpoint?param1=value1¶m2=value2
  • path parameters: baseURL/enpoint/path_parameter

Response status code

  • 1XX: informational responses
  • 2XX: successful responses
  • 3XX: redirection messages
  • 4XX: client error responses
  • 5XX: server error responses

Frameworks

Swagger

Allow us to design, build, document and call REST API, among others.

  • Editor online
  • Swagger UI: allows us to call our APIs.
