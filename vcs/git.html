Workflow

Usual workflow

  1. Create a repository with a branch called main.
  2. Create a branch called develop and copy the branch main.
  3. Create branches called features and copy the content from develop.
  4. Work using those feature branches.
  5. Once a development is finished, merge the feature with develop and start a new feature.
  6. When a release is ready, merge develop with main.

Workflow when a bug is detected in main

  1. Create a new issue branch and copy the branch main.
  2. Work using this issue branch.
  3. Once the bug is solved, merge the branch issue with main and develop.
